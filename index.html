See documentation

Our vision is to ensure that every individual on the planet is recognised, protected and provided for from birth.

A new model for civil registration

Increases birth and death registration rates

  • Active registration at the community level to avoid prohibitive travel and costs
  • Uses networks of trusted registration agents within the community
  • Timely notification of births and deaths from hospitals

Makes civil registration easy

  • Simplified application process that includes reduced data collection requirements
  • Customer-centric registration services, including application status updates
  • Automated validation for accurate data collection

Drives efficiencies for government service delivery

  • Works with other e-government systems as a foundation for effective ID management
  • Real-time operational data that supports increased accountability and performance management
  • Digital first to reduce manual processes and paper archives

OpenCRVS process

While every country will have a slightly different process that uses different actors, OpenCRVS supports key elements of a standardised process that reflects best practice and drives efficiency.

Notify a vital event

This is an optional step to collect a minimal set of vital event data that can be used for (i) planning and (ii) follow-up purposes. In OpenCRVS we call it an “in-progress” application because action is required to complete it. You can receive an “in-progress” application from the OpenCRVS application (as an incomplete application) as well as a number of external sources including:

  • Digital health systems e.g. DHIS2 or OpenSRP
  • Messaging apps e.g. SMS, USSD, Whatsapp etc. (self-notification)
  • Refugee registration systems
Learn more

Declare a vital event

This is the completion of a vital event registration form. In OpenCRVS we call this an "application" as it is more familiar to service providers - all mandatory information + supporting documents required to register the vital event. This can be completed:

  • At the community level on a tablet or mobile device
  • In an office on a PC, laptop or any other available device
Learn more

Validate and register a vital event

In OpenCRVS, this is ideally done in one step. In the “Ready to Review” workqueue, the Registrar (or Registration Agent) can find all applications sent from the field, review them alongside supporting documents, and then either reject or register them.

Learn more

Issue a certificate

Once the record has been registered, the certificate is “Ready to Print” and it appears in the workqueue of the same name, ready to be printed with digital signatures and collected. If digital signatures aren’t allowed it can be printed in advance and signatures added by hand.

Learn more

Use civil registration data

Civil registration data is incredibly valuable and OpenCRVS helps you realise this through:

  • Vital statistics export: export your civil registration data in a standard format that can be used in a vital statistics system
  • Operational data: this lets registration managers understand what is working and what is not and provide targeted support and resources
  • Headline registration rates: in real-time, see key rates required for ongoing performance management at the sub-national, national and global level, including SDG indicators that can be taken from civil registration data.
Learn more
